Storytelling with GIFs


Graphical Interchange Format (GIF) has become one of the most popular image formats on the web due to how easily it can be shared and how simple yet effective it can be.

My Role

This project has spanned over two school quarters. In the first quarter, I mainly focused on the front end prototyping design and in the second quarter, I helped program the backend.

Length of project

September 2019 - March 2020

Tools used

Adobe XD | React Native

Phase 1

Initial Research


There are many different GIF creator tools but they lack specific features that we feel are necessary to tell a story. We wanted to make sure this tool was not overwhelming due to too many options when creating a storytelling GIF.

Therefore, the main purpose behind this project is to:


- create a tool that allows novice storytellers to create GIFs that can be combined with texts and shared on many different platforms easily

- our tool was designed to limit design options so the user can focus on the story itself

Understanding the GIFs Landscape


Before designing our tool, we had to understand the different ways GIFs are used and shared. In order to do so, we brainstormed and classified GIFs into four different types that were most common:


1. Moments - no text or background added to GIF, simple footage of the topic

2. Illustrations - moving graphics or drawings, usually some sort of artwork

3. Explainers - show data in a moving format usually, graphs and charts

4. Memes - add humor or cultural reference to GIF

Takeaways


Our observations showed that many people who use GIFs on social media use them for humor which explained the abundance of “Meme” GIFs. However, there are still many examples of people sharing GIFs on various platforms to convey information. Moving forward, we decided to prototype stories based on existing platforms and implement characteristics that lend themselves to storytelling. Since brevity is the main goal for our tool, we felt some further research was needed to decide what the proper constraints would be e.g. how long to make each frame, number of frames, etc.


In order to make the most efficient tool to tell stories in a concise but effective matter, we did further research to understand which GIFs were most successful. For storytelling, we felt 8 seconds with 5 frames maximum were the best constraints because:


- enough time with a loop to get the story’s point across without losing the viewer’s attention

- limit the user from spending time on unnecessary things such as figuring out number of frames, etc.

Prototyping


To begin prototyping, we iterated through several different paper prototypes of mobile application designs. We experimented with:


- drop down boxes

- 5 W’s (who, what, when, where, and why)

- caption followed by picture or vice versa, etc.

User Feedback


Some user observations and feedback include:


• User was sometimes confused where to click

• guide the user to know that the next however many frames need to be filled out in order for the GIF to be made

• Can you make a GIF with less than 5 frames?

• Is there a time limit to how long each frame should be?

• user should decide the length of frame and that 1.5 secs is too fast in order to absorb all the information

• include a walkthrough or pop messages to show how to use platform

Final Prototype


For the final prototype, we took the user feedback and observations into considerations and made final touches on our mock up design. We were able to create a GIF storytelling tool that:


• includes a user library for previously created GIF stories

• 5 W’s to guide users when creating a story

• Allows for caption or picture first

• Shareable even if GIF is less than 5 frames

• Shareable on many different platforms
